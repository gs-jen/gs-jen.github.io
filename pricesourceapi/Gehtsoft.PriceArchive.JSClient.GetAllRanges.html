public method Gehtsoft.PriceArchive.JSClient.GetAllRanges

Brief

Returns the range of the price data available for each instrument

Declaration
JavaScript
Gehtsoft.PriceArchive.JSClient.GetAllRanges (callback)

Parameters
callback

The callback will be called when the method is complete and must be a function accepting an object which has the following properties:

Code

The error code or 0 if the operation was successful

Error

The error message in case the operation wasn't successful

Ranges

An array of the objects representing instruments and their ranges.

See below for the range object description.

Each range is represented as the object with the following properties:

Instrument

The name of the instrument

Timeframe

The name of the timeframe for which the data is available.

Note: if a group of timeframes has the same range, e.g. 1m, 5m, 15m and so on, only the shortest time frame will be represented in the list.

E.g. if the answer consists of 1m and 1h timeframes only, that means that all timeframes longer than 1m and shorter than 1h (1m, 5m, 15m, 30m) will have one range of the data, and all timeframes longer than 1h (1h, 2h, 6h, 12h, 1d...) will have the other range.

From

The date and time the price data is available from.

To

The date and time the price data is available to.

Details

Example:

Get the range of the data available

 client.GetAllRanges(function (result) {
     if (result.Code == 0) {
         for (let i = 0; i < result.Ranges.length; i++)  {
            let range = result.Ranges[i];
            console.log(range.Instrument, range.Timeframe, range.From, range.To);
         }
     } else {
         console.log("Can't get instrument data");
         console.log(result.Message);
     }
});
