Namespace KendoUI.DataRequest

In order to provide data to Kendo controls that relies (such as x-autocomplete-edit, x-select or x-grid on the server data:

1) Add KendoUI.DataRequest package from the private Gehtsoft repository to the project.

2) For each kind of the data, implement an action that takes DataSourceRequest object and returns a json object with the data.

3) The json object that is returned by the method must consist of the following properties:

Name

Type

Description

total

int

Total number of the records in the whole filtered set. The value is used when paging is requested to show proper page bar.

errors

string

Errors (if any) happened during getting the data.

data

object[]

An array of the filtered and paginated data.

The data for the x-autocomplete-edit and x-select tags must consist of two properties Id with the data identifier and Value with the text to display in the control.

The data for the x-grid must follow the schema defined in the control using x-server-field tag.

See below an example of the implementation of the data getting method on the base of EF database access.

public IActionResult AutocompleteAction(DataSourceRequest request)
{
    string mask = null;
    if (request.Filter?.Arguments != null & request.Filter?.Arguments.Length > 0)
        mask = request.Filter.Arguments[0].Value;
    try
    {
        int count = mDao.CountGoods(mask);
        var goods = mDao.GetGoods(mask, request.Skip, request.Take);
        object[] r = new object[goods.Count];
        for (int j = 0; j < goods.Count; j++)
            r[j] = new {Id = goods[j].ID, Value = goods[j].Name};
        return Json(new {total = count, data = r});
    }
    catch (Exception e)
    {
        return Json(new {errors = "Can't get data"});
    }
}
