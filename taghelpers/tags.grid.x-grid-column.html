tag x-grid-column in Grid

Brief

The definition of a grid column.

Declaration
<x-grid-column title="text" field="text" format="text" template="text" sortable="true|false" filterable="true|false" class="text" row-class="text" header-class="text" width="text" sort-by-default="no|asc|desc" can-unsort="true|false" />

Parameters
title

The title to be displayed in the grid header.
field

The associated field.
format

The format. see Globalization in Kendo UI documentation for more details about formatting.
class

The class to be applied to the column.

The option can be used, for example, to set text alignment or to hide column on lower resolution using hidden-* bootstrap tag.
row-class

The class to be applied to column in data rows.
header-class

The class to be applied to column in header data rows.
width

The width of the column in percents or in pixels.
sortable

The flag indicating whether the column is sortable.
sort-by-default

The flag indicates sort order by default. The attribute can take one of the following values: no, asc, desc
can-unsort

The flag indicating whether data can be un-sorted.

Details

If the data inside of the column requires special formatting use a template (see x-script).

Template: Generate link to the edit of the item inside of the column [show]

Template: Generate link to the edit of the item inside of the column [hide]

<x-grid-column title="Name">
    <x-script role="template" template-parameter-name="m">
        <script>
            return '<a href="@Url.Action("Edit")?id=' + m.ID ! +'">'
                + m.Name
                + '</a>';
        </script>
    </x-script>
</x-grid-column>

To define filtration, use two scripts with custom role names: filter-operators and filter-ui. The first is used to limit the number of filter operators that may be used to define the filter and the second is used to provide a custom control to enter a filter value.

See more about filter ui here: Columns filterable UI.

See more about operators here: Filterable operators.

Template: Generate link to the edit of the item inside of the column [show]

Template: Generate link to the edit of the item inside of the column [hide]

<x-grid-column title="Good" field="@(nameof(GridTransactionModel.GoodName))" sortable="true" can-unsort="false" width="60%" sort-by-default="@(Model.SortOrder == nameof(GridTransactionModel.GoodName) ? (Model.SortDir == "asc" ? KendoSortOrder.asc : KendoSortOrder.desc) : KendoSortOrder.no)">
   <x-script script-role="byname" script-role-name="filter-ui">
       <script>
           var goods = @Html.Raw(Model.Goods.ToJScript());
           e.kendoDropDownList({
               dataValueField : '@(nameof(Good.Name))',
               dataTextField : '@(nameof(Good.Name))',
               dataSource : goods,
               optionLabel: 'Select...'
           });
       </script>
   </x-script>
   <x-script script-role="byname" script-role-name="filter-operators">
       <script>
           { string : { eq : "Equals", neq : "Not Equals"} }
       </script>
   </x-script>
</x-grid-column>

To define editing, use two scripts with custom role names: editable and editor.

See more about filter ui here: columns.editable.

See more about filter ui here: columns.editor.

To define a custom column, use script byname custom. The script content is not to be expected a function body, it will be placed "as is" into the array of the columns during the configuration. For more teails about column configuration see columns.editor.
