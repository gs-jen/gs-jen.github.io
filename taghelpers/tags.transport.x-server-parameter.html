tag x-server-parameter in Transport

Brief

Declaration
<x-server-parameter operation="read|create|update|destroy" parameter-name="text" value="object" />

Parameters
operation

The name of the operation to which the parameter is set. You can omit the parameter to apply parameter to all operations.
parameter-name

The name of the parameter. You can omit the parameter name and then provide completely custom logic in the associated event.
value

The constant value to assign.

Details

There are three options how parameter may be defined:

  • If parameter-name and value is set - the parameter with the specified name and value will be added.

    Example [show]

    Example [hide]

    <x-server-parameter operation="read" parameter-name="test" value="123"  />

  • If parameter-name is set and a dataBound script is added to the tag - the parameter with the specified name will be added. The value will be taken by executing the function with the body specified in the x-script. The function has no parameters.

    Example [show]

    Example [hide]

    <x-server-parameter operation="read" parameter-name="test" >
 <x-script script-role="databound">
  <script>
   return $('#grid').data('kendoGrid').dataSource.page();
  </script>
 </x-script>
</x-server-parameter>

  • If parameter-name is not set and a dataBound script is added to the tag, the code specified in the x-script tag will be executed. The code can access two values: data - the object that consists of other parameter and operation which is the name of the operation.

    Example [show]

    Example [hide]

    <x-server-parameter>
 <x-script script-role="databound">
  <script>
   if (operation !=='read' & data.models !== null)
       return { models: kendo.stringify(options.models) };
  </script>
 </x-script>
</x-server-parameter>

The tag may be used inside of x-server-transport tag only.
