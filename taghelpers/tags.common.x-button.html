tag x-button in Common

Brief

A button

Declaration
<x-button role="submit|button|link|action" primary="true|false" id="text" disabled="true|false" href="text" target="text" label="text" tool-tip="text" sprite="text" />

Parameters
role

The button role.

The attribute can take one of the following values: submit, button, link, action (call script)
primary

The flag indicating whether the button is a primary button.
id

The button identifier.
disabled

The flag indicating whether the button is disabled.
href

The URL to navigate for link buttons.
target

The target where to navigate for link buttons (_self by default).
label

The button text.
tool-tip

The button tool tip.
sprite

A sprite to display before the icon label.

See also Kendo UI icons and Bootstrap glyphicons for available icons.

Details

All attributes described in x-base base tag can also be applied to this tag.

The control supports following event scripts: click.

Note that the button is not a form control and aren't processed using form layout engine. To properly layout buttons inside the form you can either put buttons into x-form-group control or use x-row and x-col tags for layout buttons manually.

What If Spites Aren't Displayed

Add following styles to the page stylesheet:

Style [show]

Style [hide]

.k-sprite {
    text-indent: 0;
    font-size: 16px;
    height: 24px;
    line-height: normal;
}

See also Button Overview.
