tag x-scheduler in Scheduler

Brief

The scheduler.

Declaration
<x-scheduler id="text" date="DateTime" editable="true|false" start-time="DateTime" end-time="DateTime" max-time="DateTime" min-time="DateTime" time-zone="text" major-tick="number" minor-tick-count="number" show-work-hours="true|false" selectable="true|false" height="number" var="text" tasks="IEnumerable<SchedulerTask>" />

Parameters
id

The identifier of the scheduler control.
date

The initial date displayed.
editable

Either false (default value) to make the whole control non-editable or nothing.

To make the control editable use x-scheduler-editable tag.
start-time

The start time of the week and day views.
end-time

The end time of the week and day views.
min-time

The minimum date which can be selected via the scheduler navigation.
max-time

The maximum date which can be selected via the scheduler navigation.
time-zone

The timezone which the scheduler will use to display the scheduler appointment dates.
major-tick

The number of minutes represented by a major tick.
minor-tick-count

The number of time slots to display per major tick.
show-work-hours

If set to true the view will be initially shown in business hours mode.
selectable

If set to true the user would be able to select scheduler cells and events.

To track selection you can either use change event of the control or attack a click event to elements with 'k-event' style in dataBound event.
height

Height of the scheduler in pixels.
var

The variable name to store a kendoScheduler object.
tasks

A local definition of the data source for a scheduler.

The value must be an instance of IEnumerable<SchedulerTask>

Details

The scheduler can display various calendar and time line views with the events assigned to day and/or specific time of the day.

Use one or more x-scheduler-view tags to configure scheduler views.

The control throws change and dataBound events.

For the templates that may be defined the in the object see x-scheduler-view tag.

To provide the data use either tasks attribute or add x-server-datasource tag to connect the control to remote data.

To properly provide the data for the scheduler, the application must implement the action that takes DataSourceRequest parameter, an two optional DateTime parameters in case server-filtering attribute of the transport is set and which returns an array of the task event objects as json in the datafield of the return object.

To return a error, just return a json object which has a single string field named errors.

public IActionResult SchedulerData(DataSourceRequest rq, DateTime? start, DateTime? end)
{
    try
    {
        IEnumerable<SchedulerTask> tasks = Dao.GetTasks(start, end);
        return Content(JsonConvert.SerializeObject(new {data = gDefaultTaskView},
                                                   Formatting.None,
                                                   new RecurrenceRuleJsonConverter(),
                                                   new RecurrenceExceptionJsonConverter()),
                       "application/json" );
 
    }
    catch (DaoException e)
    {
        return Json(new {errors = "Internal error"});
    }
}
