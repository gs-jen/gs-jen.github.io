tag x-validate in Form

Brief

A validation rule for a form control or a whole form.

Declaration
<x-validate error="text" is-expression="true|false" cancel-next="true|false" />

Parameters
error

The error text to show if the validation is failed.
is-expression

The flag indicating whether the content of the tag is a boolean expression or a complete function body (default).
cancel-next

The flag indicating whether next validators for the same validation target must not be executed in case of this validation is failed.

Note that the validators are executed in the order as they are defined.

Details

The tag consists of script in the form of a boolean expression or a function body that returns a boolean value.

The code may use two special variables:

value is the value of the control being validated.

reference is the function that takes one parameter name and returns the value of the control with such name. The name is the name of the control or the value of the in-validation-name attribute of the control if such attribute is set. This function may be used when the whole form is validated (e.g. x-validate tag is located right inside the x-form tag rather than a inside a control tag.

The content of the tag will be enclosed into the following function definition function (reference, value) { tag content }

For better IDE compatibility the whole content of the tag may be enclosed into <script> tag.
