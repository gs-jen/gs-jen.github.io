tag x-form in Form

Brief

The form.

Declaration
<x-form container-width="number" action="text" method="post|get|ajax|ajaxManual" title="text" mode="inline|horizontal|twocolumn|custom" id="text" for="modelExpression" name-prefix="text" label-column-width="number" control-column-width="number" body-width="number" block="true|false" />

Parameters
container-width

The width of container in bootstrap columns.
action

The URL to post or get form data to.
method

The form post method.

See below for ajax form discussion.

The attribute can take one of the following values: post, get, ajax or ajaxManual
title

The form title.
mode

The form layout mode.

The attribute can take one of the following values: inline, horizontal, twocolumn, custom

In inline mode all the controls are placed in one line. This mode is useful for filters over grids.

In horizontal mode all the controls are located inside of explicitly defined rows and columns, one control in a column, with labels above the control. row must be always declared. If column is not declared, a column with maximum width is automatically created.

In twocolumn mode the labels are located at the left side of the form and the control are located in the right side of the form, one control in a row. The rows don't have to be declared implicitly.

In case of custom mode is used, an implementation of layout engine must be set in layout-engine property.
id

The form identifier.
for

The ModelExpression of the form response object.

If this attribute is not used, the form will behave as a regular ASP.NET form, where the response object and the form page model object has the same type.

Using this attribute allows the developer to separate the model and the response object in order to keep the form supplementary information and the form input data separately or, say, to make multiple forms on one page.

The attribute MUST be used for any from that consists of embedded forms.
label-column-width

The width of the label column for twocolumn layout. The value is the bootstrap columns (1..12)
control-column-width

The width of the control column for twocolumn layout. The value is the bootstrap columns (1..12)
body-width

The width of the form body in the bootstrap columns (1...12).
block

The flag indicating whether the form must have a k-block style.

Details

Form Validation

To show the server-generated form errors use x-form-errors tags.

To provide the client-side validation use x-form-errors tag with support-on-client-validation attribute and then provide x-validate tags for each control and for the form in whole.

If client side validation is enabled, the form be automatically validated before sending to the server.

Protecting Changed Data

To prevent buttons and links inside the form from navigating out of the form when the form data is changed use x-link-guard tag.

Ajax Forms

Ajax forms uses ajax methods instead of simply posting the form as a query string to the server.

Ajax form has some limitations, for example, all the control names must be a legal name for a javascript object property name, i.e. names such as "a.b" or "a[0]" which are supported for regular forms aren't allowed.

If ajax method is used, the form is posted via ajax and as a Json object. No navigation out of the page which consists of the form happen in this case.

If ajaxManual method is used, no action is taken by default. You can use library Java Script functions to get the values from the server (tagHelper_getForm) or from the local storage (tagHelper_loadForm), to post values to the server (tagHelper_ajaxPost) or to save the values locally (tagHelper_saveForm) or even simply get form as javascript object using tagHelper_formToJson call.

Form events

The form may have two event handler: submit and error.

All attributes described in x-base base tag can also be applied to this tag.
